1. Contexte et objectifs du projet

J'ai developpe une application web full-stack nommee Goldorak DB dans le cadre de ma formation de Developpeur Web et Web Mobile a l'ecole La Plateforme (Martigues). Ce projet personnel vise a demontrer l'ensemble des competences acquises durant le cycle de formation, en couvrant l'integralite du cycle de developpement d'une application web moderne.

L'application permet de gerer l'univers de la serie culte Goldorak (UFO Robot Grendizer) a travers une base de donnees interactive, securisee et immersive. Elle propose une interface de gestion complete pour :

  • Personnages : nom FR/JP, faction, role, age, planete d'origine, description
  • Robots : nom FR/JP, pilote, type, hauteur, poids, description
  • Armes : nom FR/JP, robot associe, puissance, frequence d'utilisation
  • Vaisseaux : nom FR/JP, type, pilote, faction, description
  • Episodes : titres FR/JP, numeros, dates de diffusion, resume
  • Monstres : nom FR/JP, episode d'apparition, type, taille, puissance, description
  • Utilisateurs : authentification OAuth2 (Google/GitHub) et mode invite

L'application integre des fonctionnalites avancees : CRUD complet, recherche avec debounce, pagination, export CSV, tableau de bord en temps reel, lecteur musical thematique et un easter egg (Code Konami) pour les fans de jeux video retro.

4. Competences validees

Ce projet couvre de maniere exhaustive l'ensemble du perimetre du titre professionnel "Developpeur Web et Web Mobile" et demontre ma capacite a mener un projet de bout en bout.

  • Frontend : Maquettage (Figma), developpement de composants React 19, gestion d'etat, routage, responsive design, interactions dynamiques (tooltips, modales), integration de lecteur audio, lightbox pour les images.
  • Backend : Conception d'API REST avec Express, authentification OAuth2 (Google/GitHub), gestion de sessions JWT, validation des donnees, logging avec Winston.
  • Base de donnees : Modelisation Merise (3FN), ecriture de scripts SQL, gestion des relations et contraintes d'integrite (ON DELETE CASCADE / SET NULL), requetes preparees.
  • Deploiement : Conteneurisation Docker, orchestration Docker Compose, build multi-stage (React → Nginx), gestion des variables d'environnement.
  • Securite : Protection contre les injections SQL, validation des entrees, CORS, cookies securises, mode invite.
  • Documentation : README complet, dossier professionnel structure, captures d'ecran annotees.

5. Fonctionnalites marquantes

  • Authentification OAuth2 : Connexion via Google et GitHub, couplee a une generation de JWT pour securiser les sessions API.
  • Mode invite : Acces en lecture seule sans configuration de compte, ideal pour les demonstrations.
  • Recherche avancee : Barre de recherche avec debounce (300ms) declenchee apres 2 caracteres, sur plusieurs champs.
  • Pagination : Selecteur de 10, 20, 50 ou 100 elements par page, avec boutons de navigation stylises.
  • Tooltip de description : Affichage des descriptions longues dans une modale au clic, avec overlay opaque et fermeture automatique.
  • Lecteur musical : Playlist de 4 titres emblematiques de Goldorak, avec controle lecture/pause et selection directe.
  • Easter Egg : Code Konami (↑ ↑ ↓ ↓ ← → ← → B A) declenchant une surprise dans l'application.
  • Export CSV : Telechargement des donnees de chaque entite au format CSV.

6. Modelisation de la base de donnees

La base de donnees a ete concue selon les regles de la Troisieme Forme Normale (3FN) afin d'eliminer toute redondance et garantir l'integrite des donnees. Le schema comprend 7 tables :

  • personnages : id, nom_fr, nom_jp, faction, role, age, planete_origine, description
  • robots : id, nom_fr, nom_jp, pilote_id (FK), type_robot, hauteur, poids, description
  • armes : id, nom_fr, nom_jp, robot_id (FK), puissance, frequence_utilisation, description
  • vaisseaux : id, nom_fr, nom_jp, type_vaisseau, pilote_id (FK), faction, description
  • episodes : id, titre_fr, titre_jp, numero_fr, numero_jp, diffuse_jp, diffuse_fr, resume
  • monstres : id, nom_fr, nom_jp, episode_id (FK), type_monstre, taille, puissance, description
  • users : id, oauth_provider, oauth_id, email (UNIQUE), display_name, avatar_url, created_at, last_login

Contraintes d'integrite :

  • ON DELETE CASCADE sur armes.robot_id (suppression automatique des armes liees).
  • ON DELETE SET NULL sur robots.pilote_id, vaisseaux.pilote_id et monstres.episode_id.
  • UNIQUE sur users.email pour eviter les doublons.

7. Architecture logicielle

L'architecture est organisee selon un modele client-serveur classique, avec une separation stricte des responsabilites :

  • Frontend (React/Vite) : Composants UI, hooks personnalises, contextes d'authentification, routage.
  • Backend (Node.js/Express) : Middlewares (CORS, JSON, logging, JWT), routes API versionnees (/api/v1), couche d'acces aux donnees.
  • Base de donnees (MySQL 8.0) : 7 tables relationnelles, contraintes d'integrite, index pour les recherches.
  • Deploiement (Docker) : 3 services (db, backend, frontend) orchestres avec Docker Compose.

Cette architecture permet une maintenabilite et une evolutivite optimales. Chaque couche est testable independamment.
